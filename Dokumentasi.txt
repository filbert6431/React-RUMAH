Project ini adalah aplikasi web React + Vite untuk coffee shop yang menggabungkan dua sisi utama:

- sisi pelanggan/guest untuk melihat menu, promo, ulasan, dan informasi brand;
- sisi admin untuk mengelola dashboard, produk, order, POS, promo, customer, staff, dan membership.

Aplikasi ini dibangun dengan React 19, React Router, Tailwind CSS, dan memanfaatkan Supabase REST API sebagai sumber data utama. Untuk kebutuhan fallback, beberapa bagian juga memakai localStorage.

---

## 1. Ringkasan Proyek

RUMAH adalah aplikasi digital coffee shop yang menggabungkan pengalaman pelanggan dengan operasi admin. Tujuan utamanya adalah menyediakan antarmuka modern untuk:

- menampilkan produk dan promo secara menarik;
- mengelola penjualan melalui POS;
- melihat ringkasan bisnis di dashboard admin;
- mengelola order, produk, pelanggan, staff, dan review.

Aplikasi ini cocok untuk bisnis coffee shop yang ingin mengelola operasional harian secara sederhana namun tetap terstruktur.

## 7. Alur Data Aplikasi

### 7.1 Data dari API

Beberapa halaman mengambil data melalui service di src/Services:

- Produk diproses oleh src/Services/Products.js
- Order diproses oleh src/Services/Orders.js
- Promo diproses oleh src/Services/Promo.js

### 7.2 Fallback ke localStorage

Jika API tidak tersedia atau gagal, aplikasi memakai fallback dari localStorage dan data JSON untuk menjaga kelancaran UI.

### 7.3 Alur Checkout POS

1. Admin menambah item ke keranjang di POS.
2. Saat checkout diproses, order dibuat.
3. Stok tiap produk yang dibeli dikurangi.
4. Perubahan stok disimpan ke state dan sumber data yang dipakai halaman produk.

### 7.4 Alur Promo Claim

1. Data promo diambil dari API atau JSON.
2. Promo ditampilkan di halaman home.
3. Saat tombol claim ditekan, sistem mengecek apakah promo aktif dan apakah kuota masih tersedia.
4. Jika valid, claim_count bertambah.

### 7.5 Pola Logika Serupa Antar Halaman

Beberapa pola logika muncul berulang di banyak halaman dan penting dipahami agar pengembangan lebih konsisten:

- Normalisasi data (normalized data)
  Data dari Supabase, JSON, atau localStorage sering dikonversi dulu ke format frontend yang seragam sebelum dipakai di UI. Contohnya produk diubah menjadi struktur dengan field seperti product_id, nama_product, harga_product, stock, dan image_url agar halaman admin maupun guest bisa membacanya dengan cara yang sama.

- Pola loading, error, dan fallback
  Hampir semua halaman yang mengambil data memakai pola state loading, state error, dan fallback data. Ini membuat UI tetap aman jika API sedang tidak tersedia.

- Pemisahan logic ke Services dan Helpers
  Logika request API tidak diletakkan langsung di page yang terlalu besar. Sebagian besar dipindah ke folder Services, sedangkan helper seperti perhitungan order dan analytics diletakkan di folder lib.

- Sinkronisasi data antar halaman
  Ada beberapa fitur yang memerlukan data yang sama antar halaman, misalnya stok produk yang dipakai di POS dan di tabel produk admin. Untuk itu, digunakan penyimpanan lokal dan state yang disinkronkan agar perubahan di satu halaman terlihat di halaman lain.

- Pola state dan UI yang konsisten
  Halaman seperti Home, Menu, POS, dan Products memakai pola yang sama: data diambil saat komponen pertama kali dimuat, lalu diproses ke state, dan ditampilkan ke UI melalui komponen reusable.

- Shared business rules
  Aturan bisnis seperti promo claim, pembatasan tombol, pengurangan stok, dan kalkulasi total order diterapkan di level halaman atau service agar logika tetap konsisten di seluruh aplikasi.

#### Contoh implementasi pola-pola tersebut

1. Normalisasi data

```jsx
const normalizeProduct = (product) => ({
  product_id: product.product_id || product.id,
  nama_product: product.nama_product || product.name,
  harga_product: Number(product.harga_product || product.price || 0),
  stock: Number(product.stock ?? product.stok ?? 0),
  image_url: product.image_url || product.image || "",
});
```

Contoh ini dipakai di halaman POS dan halaman produk agar data API dan JSON dapat dipakai dengan format yang sama.

2. Pola loading, error, dan fallback

```jsx
const [products, setProducts] = useState([]);
const [isLoading, setIsLoading] = useState(true);

useEffect(() => {
  const loadProducts = async () => {
    try {
      const data = await productsAPI.fetchProducts();
      setProducts(data);
    } catch {
      setProducts(localProducts);
    } finally {
      setIsLoading(false);
    }
  };

  loadProducts();
}, []);
```

Pola ini terlihat di halaman seperti POS dan Home ketika data API sedang tidak tersedia.

3. Pemisahan logic ke Services

```js
export const productsAPI = {
  async fetchProducts() {
    const response = await axios.get(API_URL, { headers });
    return response.data;
  },

  async updateProduct(id, data) {
    const response = await axios.patch(`${API_URL}?product_id=eq.${id}`, data, { headers });
    return response.data;
  },
};
```

Logika request API dipisahkan ke services supaya halaman tidak dipenuhi kode request yang panjang.

4. Sinkronisasi data antar halaman

```jsx
setProducts((currentProducts) => {
  const updatedProducts = currentProducts.map((product) => {
    if (product.product_id === item.product_id) {
      return { ...product, stock: product.stock - item.qty };
    }
    return product;
  });

  localStorage.setItem(STORAGE_KEY, JSON.stringify(updatedProducts));
  return updatedProducts;
});
```

Kode ini menunjukkan bagaimana perubahan stok dari POS langsung disimpan dan dibagikan ke halaman produk.

5. Shared business rules

```jsx
const isDisabled = status_promo === "Tidak Aktif" || claim_count >= claim_limit;

<Button onClick={handleClaimPromo} disabled={isDisabled}>
  {isDisabled ? "Tidak Bisa Diklaim" : "Claim Promo"}
</Button>
```

Aturan bisnis seperti promo tidak aktif atau kuota penuh diterapkan di satu tempat sehingga konsisten di seluruh UI.

## 9. Panduan Kontribusi Developer

### Praktik yang disarankan

- gunakan komponen yang reusable untuk bagian UI yang berulang;
- simpan logika request API di folder Services;
- hindari menulis logika bisnis langsung di halaman jika bisa dipisahkan ke helper di folder lib;

### Tip struktur pengembangan

- tambahkan halaman baru di folder src/pages;
- tambahkan komponen UI di folder src/components;
- tambahkan service API baru di folder src/Services;
- tambahkan helper util di folder src/lib.
